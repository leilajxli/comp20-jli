Issues Found

1. Cross-Site Scripting

The first vulnerability is Cross-Site Scripting. I performed an attack by adding an alert box script in the username field using curl. The issue is on the /submit page. My recommendation is to use a validator package. Alternatively, the developer could encode all the special characters in the user's input.


My XSS code:
curl --data "username=&score=6666&grid=[]" https://peaceful-peak-19115.herokuapp.com/submit

The solution:
var validator = require('validator');
username=username.place(/[^\w\s]gi, '');
